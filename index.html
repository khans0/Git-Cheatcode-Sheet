This page is designed for beginners to learn how to link their VS Code work to Github

Create own working branch on VS
It is important that you are working on your own branch to avoid messing up the main branch.

  • Open integrated terminal - make sure you are in the correct repository.

  • Type git branch [name-your-branch]

  • Go into your newly created working branch by then typing git checkout [name-of-your-branch]

Push your own work back to GH
This allows easy collaboration without worry.

  • Save your own work locally in your own branch using
    git add .
    git commit -m "here, you should state your changes"
  • Pull all updates from Main Github branch into your working branch. This will update your working branch and you can resolve any conflicts you may have. Use
    git pull origin main
  • Merge your working branch onto Github main branch using
    git push origin name-of-working-branch

Delete your working branch
You should delete your working branch once you have finished coding the single issue you were working on. This makes it easy to maintain and update your work.

  • You must checkout from your local working branch onto your local main branch. You will not be able to delete your working branch while you are still in it. Use
    git checkout main
  • Delete working branch using
    git branch -D name-of-working-branch

How to clone a repository from Github
This is useful when you want to see what other people are up to or are asked to collaborate on a project.

  • Open the repository on Github
  • Click on "copy SSH key"
  • Open VS Code... and that's as far as I got for this

Pull repository from Github
You should do this before starting on your own work to ensure it is up-to-date.

  • Open repository on Github
  • Open integrated terminal for that repository
  • Use
    git pull
    to pull data from Github onto your local repository
  • Enter your passphrase to securely allow data pull

Merge code that has been worked on to Github
This is to link your to-do's to the working branch update in Github

  • You must convert your relevant to-do's into issues. You can do this by opening each issue and selecting "convert to issue" in the right hand column. Make sure to select the correct repository.
  • Assuming you have first pushed your work to Github. This will create a pull request. Open and compare pull request
  • Pull the request by clicking on "pull request"
  • Then scroll down and click on the Developments tag on the right hand side of the page.
  • Here, you can add whichever to-do's are relevant to your pull request and add them to it.
  • When all are selected, merge the request by clicking "merge request".
